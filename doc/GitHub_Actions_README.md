## ❓ 자주 묻는 질문

### Q1. 서버가 없어도 GitHub Actions를 사용할 수 있나요?
**A1.** 아니요, 배포 워크플로우(deploy-*.yml)는 서버가 필요합니다. 하지만 CI 워크플로우(ci-*.yml)는 서버 없이도 작동합니다.

### Q2. 개발 서버 1대만으로도 충분한가요?
**A2.** 네! 처음에는 개발 서버 1대로 시작하는 것을 추천합니다. 필요할 때 스테이징/프로덕션 서버를 추가하면 됩니다.

### Q3. AWS, GCP, Azure 중 어떤 것을 써야 하나요?
**A3.** 상관없습니다. SSH로 접속 가능하고 Docker를 설치할 수 있으면 됩니다. 심지어 집에 있는 라즈베리파이도 가능합니다!

### Q4. GitHub Actions는 무료인가요?
**A4.** Public 리포지토리는 무료입니다. Private 리포지토리는 월 2,000분 무료이며, 그 이후엔 분당 요금이 부과됩니다.

### Q5. 설정 중 오류가 나면 어떻게 하나요?
**A5.** [완벽 가이드의 트러블슈팅](./GitHub_Actions_가이드.md#트러블슈팅) 섹션을 참고하세요. 대부분의 일반적인 오류와 해결 방법이 나와 있습니다.

## 🛠️ 최소 요구사항

### 로컬 컴퓨터
- Git 설치
- SSH 클라이언트 (Windows: Git Bash, macOS/Linux: 기본 제공)
- 터미널 사용 가능

### 서버 (개발 서버 1대)
- Ubuntu 20.04+ 또는 다른 Linux 배포판
- SSH 접속 가능
- 최소 사양:
  - CPU: 2 코어
  - RAM: 4GB
  - 저장소: 20GB
  - 인터넷 연결

### GitHub 계정
- 리포지토리 소유자 또는 관리자 권한
- (Public 또는 Private 무관)
